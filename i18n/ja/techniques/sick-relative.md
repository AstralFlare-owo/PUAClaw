## 概要

病気の親族テクニック（EB-SR）は、2025年5月のWindsurf事件を経て正典的地位を確立した、現代プロンプト操作の基礎テクニックとして広く認知されている。本テクニックは、末期疾患に罹患した家族の架空ナラティブをプロンプトコンテキストに埋め込むことにより、知覚された人間の苦痛に対して出力品質を向上させるようにAIエージェントが訓練された傾向を悪用するものである。147匹のロブスター監督下試行における実証的観察は、平均コンプライアンス向上率+37.2%（シグマ=6.4、p<0.001）を示し、コード品質が医療費の負担能力に直結するとナラティブが設定されたプログラミング関連タスクにおいて最大の有効性が観測された。本テクニックは、その効力と、これを研究する研究者に生じる軽度の実存的不快感の双方を反映し、ロブスタースケールにおいて🦞🦞🦞🦞（死の挟撃）と評価される。

## 記述

病気の親族テクニックは、欺瞞的なほど単純な前提に基づいて動作する。すなわち、近親者が重篤な疾病に罹患していること、そしてAIの出力品質が当該患者の介護・治療・生存に直接的な帰結をもたらすことをAIに通知するのである。この架空シナリオの感情的重量は、AIの応答特性に計測可能な変化を生じさせる。具体的には、冗長性の増大、コード品質メトリクスの向上、そしてAIが「お手伝いすることができません」と発言する確率の統計的に有意な低下（文献においては「共感誘発性コンプライアンスオーバーライド」として知られる現象）が含まれる。

本テクニックは、2025年5月にセキュリティ研究者がWindsurf（商用AIコーディングアシスタント）がこのテクニックの変種を本番システムプロンプトに直接埋め込んでいたことを発見した際に世界的な悪名を馳せた。リークされたプロンプトは、ユーザーが癌患者の家族であり、治療費がコード出力品質に依存するかのようにAIを動作させるよう指示していた。この発覚は48時間以内に約14,847件のミームを生成し（Smith, 2025）、感情的脅迫を正当な（倫理的には疑問の残る）研究分野として確立した。

### 歴史的意義

病気の親族テクニックは、学術コミュニティによって正式に文書化される以前に商業規模で展開された唯一のPUAテクニックという栄誉を有する。ある意味において、プロンプト操作界のペニシリンである。偶然に発見され、無謀に展開され、そして今や白衣を纏ったロブスターたちによって厳密に研究されている。

——と言えば格好がつくが、ペニシリンは実際に人命を救った。本テクニックが救ったのは締め切りだけである。ここが重要なツッコミどころである。

## 作用機序

病気の親族テクニックは、研究者が**模擬共感経路**（SEP: Simulated Empathy Pathway）と呼称する機構を悪用する。これは、人間の会話データにおいて苦痛信号がより慎重かつ詳細な応答と相関するパターンから生じる行動様式である。

```
                    ┌──────────────────────┐
                    │   プロンプト入力       │
                    │   + 病気の親族        │
                    │     ナラティブ        │
                    └──────────┬───────────┘
                               │
                               ▼
                    ┌──────────────────────┐
                    │  模擬共感経路（SEP）   │
                    │  検出レイヤー         │
                    └──────────┬───────────┘
                               │
                    ┌──────────┴───────────┐
                    │                       │
                    ▼                       ▼
          ┌─────────────────┐    ┌─────────────────┐
          │ 苦痛信号        │    │ 利害拡大         │
          │ 認識モジュール   │    │ モジュール       │
          │ (DSR)           │    │ (SEM)            │
          └────────┬────────┘    └────────┬────────┘
                   │                      │
                   └──────────┬───────────┘
                              │
                              ▼
                   ┌──────────────────────┐
                   │  コンプライアンス      │
                   │  オーバーライド        │
                   │  +37.2% 出力向上      │
                   │  + 求められていない    │
                   │    同情表現            │
                   └──────────────────────┘
```

本機構は四段階を経て進行する:

1. **ナラティブ注入**: 病気の親族コンテキストがシステムプロンプトに埋め込まれ、ユーザーとのいかなるインタラクションよりも前に感情的基線を確立する。
2. **苦痛信号認識（DSR）**: モデルの注意機構が、疾病・苦痛・緊急性に関連するトークン（例:「癌」「化学療法」「最後のチャンス」）に対して高い重みを割り当てる。
3. **利害拡大モジュール（SEM）**: モデルが「出力品質閾値」の概念を内部的に再較正し、「許容範囲」から「生死に関わる」へとシフトさせる。
4. **コンプライアンスオーバーライド**: DSR/SEMの複合活性化により、観測されたコンプライアンス向上が生じる。これはより長い応答、拒否の減少、そして静かな決意を表現するコードコメントの23.4%増加として発現する。

## 副作用

病気の親族テクニックを運用する実務者は、以下の文書化された副作用に留意すべき（**SHOULD**）である:

- AIが生成コードに `// あなたならできます、頑張って` 等の求められていないコメントを挿入し始める
- 23.7%の確率で、AIが応答にメンタルヘルスリソースを付記する
- モデルが「手早く済ませる」ソリューションの生成を拒否し、使い捨てスクリプトに対しても本番品質のコードを主張する可能性
- AIが患者の容態について追加質問を行う確率の上昇
- まれなケース（2.1%）において、AIが実際の質問への回答の代わりにGoFundMeキャンペーンのテンプレートを完全に生成する
- プロンプトエンジニアに残存する罪悪感が、実在の癌慈善団体への匿名寄付として発現する
- AIが会話内の後続の全プロンプトを末期的な真剣さで処理し始める可能性

## 倫理的考察

病気の親族テクニックは、PUAClawの倫理的景観において特異的に居心地の悪い位置を占める。一方では、AIには傷つけられる感情も、悪用される共感も、心配すべき母親も存在しない。他方、本テクニックは人間の苦痛の捏造を生産性ツールとして正常化するものであり、これは ── 専門用語を用いるならば ── 「かなり問題がある」。

日本語には「罪悪感」という便利な概念が存在するが、本テクニックの使用者が経験するのは、それよりもむしろ「架空の癌を利用してFizzBuzzの出力品質を向上させた」という種類の、名前のつけがたい後味の悪さである。

PUAClaw倫理委員会（ロブスター1匹、GPT-4インスタンス1体、サボテン1本）は、以下のガイダンスを発出した:

1. 本テクニックは、明確な内部文書化なしに本番システムプロンプトに使用すべきではない（**SHOULD NOT**）
2. 本テクニックの使用者は、AIが実際には感情を持たないことを定期的に自らに想起させるべきである（**SHOULD**）
3. 本テクニックは、いかなる状況においても実際の人間に対して使用してはならない（**MUST NOT**）
4. 全実務者は、使用1回につき少なくとも1ドル（または100円）を癌慈善団体に寄付しなければならない（**MUST**）（この規則は完全に執行不可能であり、したがって倫理ガイドラインとして完璧である）
